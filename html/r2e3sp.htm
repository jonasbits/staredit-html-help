Triggers: User Values

The following is a complete list of the user definable values that can be used with the Actions and Conditions in the scenario editor.

ALLY STATUS

This option box allows you to change a value from Enemy, Ally and Allied Victory. A description of each of these values can be found in the game’s documentation.

COMMENT

This is a text field that allows you to describe your trigger.

DESTINATION

Destination is the same as Location.

DURATION

Duration may be a qualified number or number value, measured in milliseconds. 1,000 ms equals 1 second.

LABEL

Label allows you to define a name for the leader board, which will be displayed next to the top player’s score when the leader board is displayed.

LOCATION

Locations are areas that have been defined in the Location Layer of the map. The “Anywhere” Location covers the entire map. If you create a unit at “Anywhere,” it will appear in the center of the map. All locations except “Anywhere” can be restricted to certain elevations. For more information, see Creating Locations.

NUMBER

This value can range from 1-9,999,999.

NUMBER OF UNITS

This value can range from 1-255. Choosing the option box for “All Units” will disable the ability to enter a number, and the trigger will affect all units.

ORDER

Order specifies whether the affected unit(s) will Move, Patrol or Attack. The difference between these commands is listed in your game documentation.

PERCENT

This value can range from 1-100. The value that is chosen dictates what percentage the trigger will affect. For example, choosing 10 for this value will specify 10 percent out of the total. If the total were 250, 10 percent would represent 25.

PLAYER

Player is used to specify which player or players are subject to the action or condition. Valid player values are Player 1-8, Current Player, Foes, Allies, Neutral Players, All Players, Force 1-4, and Non Allied Victory Players.

POINTS

Points can be used to track combat kills, construction or a custom score set with triggers. This is normally used with the leader board action to display a constant score for each player.

‘Total’ points are the combined total of units, buildings, kills and razings.

‘Units’ points are calculated based on the number of units that were built in the scenario multiplied by the number of points associated with a unit.

‘Buildings’ points are calculated based on the number of buildings that were built in the scenario multiplied by the number of points associated with a unit.

‘Units and Buildings’ points are the combined totals from units and buildings.

‘Kills’ points calculate the number of unit kills and factor them into a score.

‘Razings’ points calculate the number of buildings that were destroyed and factor them into a score.

‘Kills and Razings’ points are the combined totals from kills and razings.

‘Custom’ points are values that are user defined. Custom points can only be modified using Trigger Actions.

PROPERTIES

This will bring up the Unit Properties dialog box.

QUALIFIED NUMBER

This combines a number with the ‘Or More’, ‘Or Fewer’, and ‘Exactly’ qualifiers.

QUANTITY

Quantity might be a qualified number, number, or number of units value. To determine which is applicable, check the title of the dialog box that appears when you click on Quantity.

RESOURCES

The resources value allows you to choose Minerals, Gas, or Minerals and Gas.

SCENARIO

This selection brings up the standard ‘File Open’ dialog box. Choose the map that you wish to be loaded when a player finishes this scenario. Using the Set Next Scenario trigger allows you to create a fully featured campaign similar to the single player levels that are included with Starcraft or even a branching campaign composed of multiple scenarios.

SCORE TYPE

This value represents the scoring method that you wish to use in your condition or action. You can set the scoring values from the points user value.

SCRIPT

This value is the computer AI script that you wish to use. The pull-down list specifies the available AI scripts. By changing the computer AI behavior, you can, for example, instruct computer-controlled units to attack certain players.

SET

Set allows you to modify or set the value of another variable, such as a player’s score or available resources. Set can also be used to enable, disable or toggle switches and certain doodads.

SWITCH

A switch is a simple on/off variable. Using switches allows you keep track of which triggers have been used or to keep a trigger from activating several times simultaneously if multiple players meet the conditions at the same time.

SWITCH IS SET

This allows you to test whether a specific switch is in the ‘Set’ state or the ‘Cleared’ state.

TEXT or TEXT MESSAGE

Similar to the WAV File option, this allows the computer to notify you when the associated condition is achieved. Note that text messages will only appear if the affected player has Subtitles turned on in the Starcraft Sound Options Menu or if the Always Display option is checked for the action.

TIMER

Timer allows you to select a number (in seconds) for the timer to be initialized at. Once the timer has been initialized, it will count down from its position in seconds and stop when it reaches zero.

UNIT or UNITS

This value sets which unit(s) will be affected by the triggers. The different unit types are:

Any Unit – This includes all types of units including buildings, resources, power-ups, etc.

Men – In general, any non-building unit that can be created over the course of the game. This also includes critters and heroes.

Building – Any structure. This includes unique structures such as the Zerg Overmind.

Factory – Any structure that can produce units.

Specific Unit – Gives you the ability to specify a single unit type.

If you want to affect more than one specific unit type, such as both Zerg Queens and Protoss Arbiters, you will need a condition or action for each unit type.

WAV FILE

This value is the name of the file that will be played by this action. For more information on importing .WAV files, see the Adding Sound Effects section.
